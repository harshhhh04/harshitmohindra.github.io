Overview

Designed and implemented an 8-bit SAP-1 (Simple-As-Possible) educational processor using Verilog HDL to explore computer architecture from the hardware level. The processor implements the complete fetch-decode-execute cycle through modular RTL components including registers, memory, an arithmetic logic unit (ALU), and a control unit. The project was developed and verified using Xilinx Vivado simulations, providing practical experience in digital design, processor architecture, and hardware verification.

Why I Built This

While learning digital logic, I realized that understanding individual components such as logic gates, multiplexers, and registers only provided a fragmented view of how a processor actually works. I wanted to bridge that gap by building a complete system where every module interacted to execute real instructions.

The SAP-1 computer was the perfect opportunity to move beyond theory and implement a processor entirely in Verilog. Instead of treating computer architecture as a collection of textbook diagrams, I designed and integrated the fundamental hardware blocks responsible for instruction execution, data movement, arithmetic operations, and memory access. This project allowed me to experience how a processor operates at the RTL (Register Transfer Level), where timing, control signals, and module interactions become as important as the logic itself.

One of the most rewarding aspects of the project was watching the processor execute instructions through simulation. Seeing registers transfer data, the control unit sequence operations, and the ALU produce results made concepts that once existed only on paper become tangible. The experience fundamentally changed how I approached computer architecture and gave me a much deeper appreciation for the engineering behind modern processors.

Beyond completing the project, building SAP-1 established a strong foundation for studying Microprocessors and Microcontrollers, allowing me to approach later coursework with practical implementation experience instead of relying solely on theoretical understanding.

Component Details

Program Counter (PC)

Purpose

Tracks the memory address of the next instruction to fetch and execute, maintaining program flow step-by-step.

Implementation

A 4-bit synchronous counter built with D-flip-flops that increments on the rising edge of the clock when the increment signal (EP) is high. It can reset to 0000 when CLR is asserted.

RTL Logic (Verilog)

module program_counter(
    input clk, clr, cp,
    output reg [3:0] count
);
    always @(posedge clk or posedge clr) begin
        if (clr) count <= 4'b0000;
        else if (cp) count <= count + 1'b1;
    end
endmodule

Memory & MAR (RAM)

Purpose

Latches addresses from the W-Bus and provides 16 bytes of SRAM to hold code instructions and user data variables.

Implementation

The Memory Address Register (MAR) stores the active RAM address when the load address signal (LM) is low. The RAM module reads/writes data to the W-Bus based on the CE (chip enable) control wire.

RTL Logic (Verilog)

module memory_mar(
    input clk, lm_n,
    input [3:0] bus_in,
    output [7:0] ram_out
);
    reg [3:0] mar_address;
    reg [7:0] ram [15:0];
    always @(posedge clk) begin
        if (~lm_n) mar_address <= bus_in;
    end
    assign ram_out = ram[mar_address];
endmodule

Instruction Register (IR)

Purpose

Acts as the execution buffer, holding the active instruction byte during decoding and execution cycles.

Implementation

Loads data from W-Bus when LI is low. The upper 4 bits are mapped to opcode pins output to the controller, while the lower 4 bits contain the memory address operand.

RTL Logic (Verilog)

module instruction_register(
    input clk, li_n,
    input [7:0] bus_in,
    output reg [3:0] opcode,
    output reg [3:0] operand
);
    always @(posedge clk) begin
        if (~li_n) begin
            opcode <= bus_in[7:4];
            operand <= bus_in[3:0];
        end
    end
endmodule

Controller / Sequencer

Purpose

Decodes instructions and sets the control lines to direct the flow of data between all other registers.

Implementation

Utilizes a 6-stage ring counter (T1 to T6). T1-T3 manage the Fetch Cycle (load PC to MAR, increment PC, and load RAM content to IR). T4-T6 decode the opcode and assert execution-phase signals (like load accumulator or perform ALU sum).

RTL Logic (Verilog)

module controller(
    input [3:0] opcode,
    input [2:0] t_state,
    output reg [11:0] ctrl_word
);
    always @(*) begin
        case (t_state)
            3'd1: ctrl_word = 12'b010111100011; // T1: PC to MAR
            3'd2: ctrl_word = 12'b101011100011; // T2: PC Increment
            3'd3: ctrl_word = 12'b010101010011; // T3: RAM to IR
            // T4-T6 states decoded here...
        endcase
    end
endmodule

Register A (Accumulator)

Purpose

Primary accumulator register that acts as the source and destination for ALU operations.

Implementation

Loads data from W-Bus when LA is low. If EA (Enable Accumulator) is high, it writes its content onto the W-Bus, distributing it back to registers or the output panel.

RTL Logic (Verilog)

module register_a(
    input clk, la_n, ea,
    input [7:0] bus_in,
    output reg [7:0] a_out,
    output [7:0] bus_out
);
    always @(posedge clk) begin
        if (~la_n) a_out <= bus_in;
    end
    assign bus_out = ea ? a_out : 8'bz;
endmodule

Register B

Purpose

Temporary helper register storing second operands during addition and subtraction instructions.

Implementation

Loads data from W-Bus when LB is low, feeding the operands directly to the ALU without outputting back to the W-Bus.

RTL Logic (Verilog)

module register_b(
    input clk, lb_n,
    input [7:0] bus_in,
    output reg [7:0] b_out
);
    always @(posedge clk) begin
        if (~lb_n) b_out <= bus_in;
    end
endmodule

Arithmetic Logic Unit (ALU)

Purpose

Performs basic arithmetic operations (addition and subtraction) on data values.

Implementation

An asynchronous hardware block that outputs Register A + Register B when control wire SU is low, and performs A - B when SU is high. Active outputs are enabled to the bus when EU is active.

RTL Logic (Verilog)

module alu(
    input [7:0] a, b,
    input su, eu,
    output [7:0] bus_out
);
    wire [7:0] result;
    assign result = su ? (a - b) : (a + b);
    assign bus_out = eu ? result : 8'bz;
endmodule

Interesting Challenge

Initially, subtraction produced incorrect values because subtraction was computed using simple unsigned values. Corrected the design by using two's complement sign-extended values to handle sign logic properly across all states.

Output Register

Purpose

Receives arithmetic outputs to display results to external circuits or monitors.

Implementation

Loads data from W-Bus when LO is low. It continuously displays its 8-bit register state directly to output lines.

RTL Logic (Verilog)

module output_register(
    input clk, lo_n,
    input [7:0] bus_in,
    output reg [7:0] display_out
);
    always @(posedge clk) begin
        if (~lo_n) display_out <= bus_in;
    end
endmodule

Instruction Set

The processor supports a minimal but complete instruction set including LDA (Load), ADD, SUB, OUT (Output), and HLT (Halt). Each instruction is 8 bits wide — 4-bit opcode and 4-bit operand — allowing up to 16 memory addresses.

Key Takeaways

This project deepened understanding of how hardware executes software at the lowest level — from clock cycles driving state machines to how control signals orchestrate data flow across registers and the ALU.
